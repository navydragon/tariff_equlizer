# Тарифный эквалайзер РУТ/РЖД

## 1. Общий обзор

**Тарифный эквалайзер** — цифровой инструмент Центра тарифного моделирования Управления новых проектов и технологий РУТ для гибкого анализа и обоснования тарифных решений в железнодорожных грузовых перевозках. Продукт высоко оценён в ОАО «РЖД» и применяется департаментом методологии тарифообразования для обеспечения прозрачности, гибкости и экономической обоснованности стратегии тарифного регулирования.

### Назначение и задачи платформы:
- Гибкое моделирование тарифов с учетом рыночных, валютных, отраслевых факторов, структуры расходов, инвестпрограмм, уровня прибыльности клиентов.
- Быстрое сравнение сценариев, анализ последствий изменений тарифов по годам, ключевым клиентам, маршрутам и видам грузов.
- Обоснование управленческих решений и формирование прозрачных отчётов для руководства и регуляторов (ФАС и др.).
- Снижение финансовых и рыночных рисков РЖД за счет комплексного сценарного моделирования.

## 2. Ключевые пользовательские страницы

### 2.1 Эффекты решений / Эластичность спроса

- Главная "управленческая" страница для оценки покрытия дефицита и вклада индексации и отдельных решений по ключевым группам грузов и годам (2025–2030).
- Карточки показывают вклад базовой индексации и специальных мер в покрытие дефицита (в млрд ₽ и %).
- Ниже: таблица эффектов по видам грузов, параметрам решений (группировка, фильтрация, сравнение сценариев).
- Визуализация позволяет увидеть где и за счет каких мер достигаются цели (по сегментам, видам бизнеса).
- Аналитики и стратеги РЖД используют раздел для подбора и проверки возможных комбинаций тарифных решений "на лету".

### 2.2 Экономика грузов

- Глубокая аналитика расходов, себестоимости, структуры ставок и маржинальности по направлениям/видам операций.
- Позволяет по выбранным фильтрам (тип расчёта, холдинг, маршрут, вид груза, период) увидеть полную калькуляцию по годам, а также интерактивно задать сценарные индексации для каждого фактора с помощью ползунков.
- Сразу отражает обновлённую структуру расходов и прибыльности, даёт сводную и детализированную визуализацию в разных разрезах (вкладки: структура, тренды, ключевые показатели и др.).
- Применяется для внутреннего обоснования тарифных ставок, сценарных расчётов департаментов, формирования комплексных отчетов по клиентским сегментам.

### 2.3 Куб эффектов

- OLAP-куб: универсальная сводная таблица по всем моделируемым результатам (по грузам, мерам, периодам и другим срезам).
- Можно агрегировать и детализировать по различным признакам, выгружать результат в отчёты.
- Используется для топ-менеджмента/аналитиков для быстрого масштабного анализа эффективности тарифной политики во всех ключевых сегментах.

## 3. Вспомогательные и служебные страницы

- **Сводные (pivot):** Интерактивный отчётчик/конструктор для самостоятельного построения кастомных сводных таблиц/отчётов.
- **Загрузка базы/обновление справочников:** Модули для обновления исходных данных, справочников, параметров моделирования.
- **Старт, настройки, экспорт:** Стартовый экран, настройки и вспомогательные интерфейсы для администраторов и опытных пользователей, экспорт рассчитанных данных и отчётов.
- **UI-навигация, параметры:** Единый стиль навигации и управления глобальными параметрами.

---

## 4. Техническая архитектура

- Базируется на Python/Dash (Plotly Dash), что обеспечивает универсальность и реактивность интерфейса.
- Используются Dash Pages (многостраничность), компонентный дизайн и callbacks для интерактивных вычислений/визуализации.
- Справочники, маршруты, сценарии загружаются через файлы feather, excel, csv.
- Поддерживается экспорт отчётов, система фильтров и state/caching для повышения отзывчивости.
- Продукт готов к масштабированию и внедрению новых направлений.

---

## 5. Значимость результатов для РЖД и сценарии применения

- **Аргументация и защита тарифных решений:** Быстрое моделирование и доказательство последствий на конкретных цифрах.
- **Управление рисками:** Проверка стресс-сценариев и подбор компенсационных мер под реальные рыночные изменения.
- **Планирование инвестиций:** Мгновенное связывание тарифных сценариев с необходимым объёмом финансирования инвестиционных программ.
- **Клиентские сценарии:** Настройка индивидуальной калькуляции для крупных клиентов, сравнение последствий для разных сегментов, предложение оптимизации тарифов.
