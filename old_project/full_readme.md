# Полное описание платформы «Тарифный эквалайзер»

## 1. Общий обзор платформы “Тарифный эквалайзер” в контексте РЖД

“Тарифный эквалайзер” — это специализированный цифровой инструмент для моделирования, анализа и обоснования тарифных решений в сфере железнодорожных грузовых перевозок. Разработан Центром тарифного моделирования Управления новых проектов и технологий РУТ и высоко оценён специалистами Департамента методологии тарифообразования ОАО «РЖД».

### Ключевые цели и задачи платформы:

- **Гибкое моделирование тарифов:**
  - Система позволяет быстро просчитывать сценарии изменения тарифов с учетом множества факторов: рынок, стоимость сырья, валюта, различные компоненты себестоимости, доля транспорта в цене на продукцию, уровень прибыли клиента и др.
- **Аналитика для принятия решений:**
  - Программа визуализирует структуру расходов, прибыль, эффективность, влияние индексации и отдельных тарифных решений, предоставляя РЖД инструмент для доказательного принятия стратегических решений.
- **Обоснование и защита тарифных решений:**
  - Возможность аргументировать выбранную тарифную стратегию перед руководством и государственными органами, используя прозрачные сценарные расчёты и макроэкономический прогноз.
- **Управление рисками и реализация инвестиционной программы:**
  - Уникальность платформы — в интеграции с параметрами финансового плана и инвестиционной программы РЖД. Это позволяет оценивать влияние тарифных решений на общую экономику компании.

### Бизнес-контекст:

- Пользователями платформы являются экономисты, тарифологи, стратеги и менеджмент РЖД.
- Система внедрена для оптимизации тарифного регулирования, повышения прозрачности, обоснованности и адаптивности тарифной политики, что особенно важно при изменчивой рыночной ситуации.

## 2. Детализация ключевых страниц

### 2.1 Страница "Эффекты решений / Эластичность спроса"

#### Назначение страницы
Эта страница служит интерактивным инструментом для оценки влияния различных тарифных решений (в основном — индексации и специальных отдельных решений) на финансовый результат и покрытие дефицита в будущем. Главная задача — показать в динамике по годам, насколько различные меры позволяют достичь стратегических целей РЖД, и какой вклад вносят отдельные сценарии для разных групп груза.

#### Основные пользовательские элементы и сценарии:
- **Верхние карточки по годам (2025–2030):**
  - Показывают вклад индексации и специальных решений для покрытия дефицита;
  - Демонстрируют динамику;
- **Таблица/панель эффектов:**
  - Группировка по группам грузов, типам тарифных решений, группировка внутри (например, по холдингу или маршруту);
  - Показываются отдельное влияние базовых и специальных решений, итоговые значения;
  - Отдельная колонка “увеличение нагрузки” — дополнительная выручка/покрытие;
- **Визуализация (горизонтальные столбики):**
  - Мгновенно показывает, какой сегмент получает больший эффект;
  - Базовые/отдельные решения подсвечиваются разными цветами;
- **Фильтры по сегментам:**
  - Для быстрого сужения аналитики на нужных сегментах (например, группы грузов, холдинги);
- **Переключение года:**
  - Можно анализировать эффекты в разрезе по выбранному году;

#### Бизнес-логика и примеры интерактивности
- Пользователь задаёт параметры (фильтр, год, сценарий) — система сразу пересчитывает показатели во всех визуальных и табличных блоках;
- Можно визуально сопоставить, сколько даст индексация и сколько целенаправленные решения для определённых групп;
- Визуальная динамика (зелёные/красные приросты/потери) облегчает анализ;

#### Значение
Это позволяет не только быстро оценивать последствия, но и оперативно управлять тарифной политикой в зависимости от целевых ориентиров.

### 2.2 Страница "Экономика грузов"

#### Назначение
Страница для глубокой аналитики структуры расходов, формирования себестоимости и прибыльности по направлениям и видам перевозок. Она показывает из чего складывается тариф и как меняется итог при разных сценариях индексации.

#### Основные элементы:
- **Фильтрация:**
  - Тип расчета: маршрут, узел, холдинг и др.
  - Группа груза, вид сообщения (экспорт/рынок), холдинг, маршрут, вид перевозки, период (годы).
- **Блок индексации по годам:**
  - Индексация по отдельным статьям расходов на любой временной горизонт (ползунки);
- **Вкладки/разделы:**
  - Тренды, структура (визуализация расходов и показателей), ключевые KPI, панели структуры по подкритериям;
- **Основная таблица и графики:**
  - Отображают структуру и динамику расходов, прибыли, доходности по годам, эффекты изменений индексации;

#### Применение
- Эксперт может быстро задать индивидуальный сценарий, оценить последствия по всем ключевым метрикам, подобрать оптимальную комбинацию факторов для достижения финансовых целевых ориентиров.
- Универсальный аналитический инструмент для внутреннего тарифообразования РЖД и обоснования ставок.

### 2.3 Страница "Куб эффектов"

#### Назначение
Это OLAP-куб для многомерного анализа итогов моделирования.

#### Элементы:
- Навигация по агрегирующим и деталям;
- Группировка по любому признаку из предустановленных (грузы, периоды, решения и т.д.);
- Древовидный вывод — каждое значение раскрывается вплоть до детализации;
- Кнопка экспорта;

#### Практическая ценность
- Быстрый агрегированный срез для отчетности, топ-менеджмента, вышестоящих органов;
- Возможность получения полной структуры вклада эффектов по годам, решениям, сегментам;

## 3. Вспомогательные и служебные страницы

- **Стартовая** — приветственный и вводный экран, переход к основным инструментам;
- **Сводные (pivot):** — интерактивный конструктор "как в Excel", для сбора кастомных сводных отчётов;
- **Загрузка/обновление базы:** — служебные экраны для обновления исходных параметров, справочников;
- **Экспорт/аналитика/куб** — разделы для доп. аналитики, экспорта продвинутых расчетов, оценки маржинальности, сценарного анализа по особым сегментам;
- **Навигация/Параметры/Настройки** — глобальные параметры, профили, режимы пользователя и пр.

## 4. Техническая архитектура и реализация

### Основа и стек
- **Dash (Plotly Dash)** — реактивная платформа построения веб-интерфейса;
- **Dash Pages** — каждая страница — модуль/класс;
- **Python** — мощная серверная логика, расчёты, интеграция с pandas/numpy и др.;
- **Plotly, Bootstrap Components** — визуализация, адаптивный дизайн;

### Архитектурные решения
- Компонентный подход;
- Связка всех страниц через page_container;
- Callbacks — реактивные расчёты и визуализация при изменении любого фильтра или параметра;
- Данные подгружаются из файлов feather, excel, csv;
- Кэширование, state-параметры для скорости и модульности;
- Экспорт через dedicated endpoints;
- Безопасность — аутентификация (задел на роли), масштабируемость модулей;
- Производительность — векторные форматы, быстрые расчёты, пофрагментная подгрузка;

## 5. Значимость результатов для РЖД и типичные сценарии использования

### Для чего нужно РЖД
- Прозрачные сценарии и визуализация для переговоров с ФАС/правительством;
- Проверка стресс-сценариев, быстрая адаптация при изменении рынка, госрегулировании, валюты и пр.;
- Аргументированное принятие управленческих решений, быстрый расчет потребности в компенсациях или изменениях тарифов;
- Оценка маржи и прибыльности по клиентам и сегментам для кросс-субсидирования и контроля экономики;

### Типовые пользовательские сценарии
1. **Обоснование тарифов для ФАС:** наглядный расчет и демонстрация вклада каждого решения/сценария;
2. **Адаптивность:** быстрое изменение параметров сценария — новый тариф тут же отражается во всех таблицах и графиках;
3. **Планирование капитальных расходов:** связывание требуемой индексации с инвестиционной программой;
4. **Клиентский аудит:** индивидуальный сценарий для крупного клиента (производная себестоимость, оптимизация);
